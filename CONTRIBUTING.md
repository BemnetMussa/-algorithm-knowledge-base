# Contributing Guide

Thanks for contributing to the Algorithm Knowledge Base.

## Quick checklist
1. Claim your topic first (issue or claim table in `README.md`).
2. Create your note file in the correct folder.
3. Follow the required section order from `README.md` (or `AGENTS.md`).
4. Add at least one reusable template with clear inputs/outputs.
5. Add concept-check questions + answers.
6. Add LeetCode/Codeforces practice links (no answers needed there).
7. Open a pull request.

## Writing expectations
- Prefer clear, practical wording over formal textbook style.
- Include small ASCII diagrams when they improve understanding.
- Add short, high-signal comments in code (invariants, edge cases, TC/SC hints).
- Keep content beginner-friendly but not oversimplified.

## Template conventions
Before each code block, include:
- `Use when:`
- `Input:`
- `Output:`

This makes templates easy to scan and reuse quickly.

## Naming and paths
- Use lowercase and hyphens for filenames.
- Use root topic folders like `graph/`, `linked-list/`, `heap/`, `recursion/`, etc.

## Example path
- `heap/heap.md`
- `recursion/recursion.md`
- `linked-list/singly-linked-list.md`
